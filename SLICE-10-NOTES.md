## SQL aggregations (golden-pinned)

All three are `public const string` on `SqlComfortBudgetFetcher` so
the golden-string tests can lock the shape without touching DI.

### 1. Hours outside zone

```sql
SELECT COUNT(*)
  FROM dbo.fv_comfortscores_at_cursor(@asOf)
 WHERE BucketTime >= @start
   AND BucketTime <= @end
   AND Score < @threshold;
```

### 2. Worst calendar cell

```sql
SELECT TOP 1 [Date], DayOfWeek, MeanResidual, MaxAbsZscore, Pattern
  FROM dbo.fv_dayprofiles_at_cursor(@asOf)
 WHERE [Date] >= @startDate
   AND [Date] <= @endDate
 ORDER BY MeanResidual ASC, [Date] DESC, DayOfWeek ASC;
```

**Tie-breaker (judgment call per #12):** most-recent `Date` wins
when two days share the same `MeanResidual`. The spec doesn't
prescribe a tie-breaker; "most recent" matches the dashboard's
calendar-this-week framing.

### 3. 7-day comfort trend

```sql
SELECT DATE_BUCKET(DAY, 1, BucketTime)                          AS Day,
       CAST(MIN(Score) AS DECIMAL(5, 2))                        AS MinScore,
       CAST(MAX(Score) AS DECIMAL(5, 2))                        AS MaxScore,
       CAST(AVG(CAST(Score AS DECIMAL(8, 4))) AS DECIMAL(5, 2)) AS MeanScore,
       COUNT(*)                                                  AS SampleCount
  FROM dbo.fv_comfortscores_at_cursor(@asOf)
 WHERE BucketTime >= @start
   AND BucketTime <= @end
 GROUP BY DATE_BUCKET(DAY, 1, BucketTime)
 ORDER BY Day ASC;
```

`AVG(CAST(Score AS DECIMAL(8, 4)))` is the pattern T-SQL needs to
keep AVG output stable at `DECIMAL` precision; the outer `CAST` then
rounds back to the `DECIMAL(5, 2)` shape of the source column so the
client reader can use the same accessor for all three score columns.

## How to validate (live)

```bash
# Bring the stack up.
docker compose up -d --wait

# Default threshold (70), default window (7 days).
curl -s http://127.0.0.1:8080/api/comfort/budget | jq .

# Compare hours-outside-zone with threshold raised to 80.
# (AC: "Changing COMFORT_DISCOMFORT_THRESHOLD to 80 increases the
#  hours outside zone count.")
COMFORT_DISCOMFORT_THRESHOLD=80 docker compose up -d --force-recreate web
curl -s http://127.0.0.1:8080/api/comfort/budget | jq .hoursOutsideZone

open http://127.0.0.1:8080/   # macOS
```

## Judgment calls

1. **Worst-cell tie-breaker.** Spec leaves this unspecified; chose
   `ORDER BY MeanResidual ASC, [Date] DESC, DayOfWeek ASC` (most-recent
   date wins). Documented in the SQL golden-string test.
2. **Empty windows return 200, not 404.** A 7-day window before the
   schedulers have warmed up legitimately has no rows. Returning 404
   would force the dashboard into the error path; 200 with
   `{hoursOutsideZone: 0, worstCell: null, trend: []}` lets the panel
   render its "no data yet" state inline.
3. **Trend sparkline uses pure SVG, not Plotly.** Single-shot static
   render; no interaction needed; smaller wire footprint; no extra
   bundle. Plotly stays available for the Explorer chart where pan/zoom
   matters.
4. **Window is `[cursor - windowDays, cursor]` with both endpoints
   inclusive on the SQL side.** `BucketTime` is hour-aligned; the
   cursor lands at finer resolution; inclusive `<=` on both ends
   avoids dropping the current hour from the budget.
5. **No "windowed()" cursor helper.** Slice 1's `CursorSnapshot.Windowed`
   returns `(AsOf - duration, AsOf)`, which would have worked, but the
   ComfortBudgetReadService computes the window explicitly so the
   pinned-SQL tests can verify start/end parameters directly without
   round-tripping through the helper.
